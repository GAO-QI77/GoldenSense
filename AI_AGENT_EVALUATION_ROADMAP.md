# AI Agent 评估问卷与改进路线图

**文档版本**: 1.0  
**评估人**: 微软 AI Agent 开发者 (Persona)

## 1. 技术架构 (Technical Architecture)

| ID | 问题描述 | 评估标准 | 测试方法 | 预期指标 | 优先级 |
|----|----------|----------|----------|----------|--------|
| T1 | **数据源一致性设计**：Streamlit 看板与 FastAPI 直播平台目前各自独立抓取数据，是否存在数据不一致风险？ | 是否实现了单一数据源（Single Source of Truth）架构？ | 同时刷新两个平台，对比同一时间点的金价与指标数据。 | 数据偏差 < 0.1% | **P0** |
| T2 | **模型推断解耦**：前端展示层（Streamlit/FastAPI）是否与 AI 推理层（Stacking Model）完全解耦？ | 推理逻辑是否封装为独立微服务？前端是否仅通过 API 调用？ | 检查代码引用，确认前端是否直接 import 了模型文件或包含硬编码逻辑。 | 完全解耦，通过 REST/gRPC 通信 | **P0** |
| T3 | **数据抓取稳定性**：核心数据依赖 `yfinance`（非官方爬虫库），是否具备高可用的容灾方案？ | 是否接入了备用数据源（如 Alpha Vantage, Polygon）？是否有自动切换机制？ | 模拟 yfinance 接口超时或封禁，观察系统是否自动切换或报错。 | 切换时间 < 5s，服务零中断 | **P1** |
| T4 | **状态管理机制**：FastAPI 服务中的 `market_cache` 使用全局变量存储，在多进程部署下是否安全？ | 是否使用了 Redis 或 Memcached 等分布式缓存系统？ | 启动多个 Worker 进程，观察缓存数据是否同步。 | 数据在所有 Worker 间强一致 | **P1** |
| T5 | **依赖管理规范**：项目是否存在未锁定的依赖版本或冗余库？ | `requirements.txt` 是否精确锁定了版本？Docker 镜像大小是否优化？ | 运行依赖扫描工具，检查构建体积。 | 镜像体积 < 500MB，无冲突依赖 | **P2** |

### 改进措施 (Technical)
*   **立即执行**: 建立 Redis 缓存层，统一 Streamlit 和 FastAPI 的数据读取来源。
*   **短期计划**: 将 `stacking_model.py` 封装为独立的推理 API 服务，移除 `pro_dashboard.py` 中的模拟推断逻辑 (`run_live_inference`)。
*   **长期规划**: 采购官方金融数据 API，将 `yfinance` 降级为兜底方案。

## 2. 功能实现 (Functional Implementation)

| ID | 问题描述 | 评估标准 | 测试方法 | 预期指标 | 优先级 |
|----|----------|----------|----------|----------|--------|
| F1 | **AI 预测真实性**：当前展示的“预测概率”与“波动率”是基于规则的模拟值还是真实模型输出？ | 代码中是否调用了 `stacking_model.predict()`？ | 代码审查 `pro_dashboard.py` 的 `run_live_inference` 函数。 | 必须调用真实训练好的模型文件 | **P0** |
| F2 | **实时数据真实性**：直播平台的 SSE 推送数据是否包含人为添加的随机噪声？ | 后端推送的数据是否完全源自真实市场 tick 数据？ | 检查 `server.py` 中的 `event_generator` 逻辑。 | 移除所有 `random.uniform` 噪声 | **P0** |
| F3 | **新闻分析深度**：新闻模块是否仅进行了简单的关键词匹配，还是使用了 NLP 模型进行情感分析？ | 是否集成了 BERT/FinBERT 等模型对新闻进行打分？ | 输入一条负面新闻，检查系统给出的情感评分是否准确。 | 情感分类准确率 > 85% | **P1** |
| F4 | **历史回溯功能**：用户能否查看模型过去 30 天的预测记录与真实行情的对比（Backtest）？ | 是否有持久化存储（Database）记录历史预测值？ | 访问历史回溯页面，检查数据的连续性和完整性。 | 数据完整无缺失 | **P2** |
| F5 | **异常检测告警**：当市场出现闪崩或暴涨时，Agent 是否能主动推送紧急告警？ | 是否具备阈值触发机制和多渠道通知能力（Email/SMS）？ | 模拟价格剧烈波动数据，观察告警触发情况。 | 延迟 < 1分钟 | **P2** |

### 改进措施 (Functional)
*   **立即执行**: 移除 `server.py` 中的 `random` 噪声生成代码，改为真实数据推送或保持最后已知价格。
*   **短期计划**: 对接 `stacking_model.py` 到 FastAPI 后端，提供真实的 `/api/predict` 接口。
*   **长期规划**: 引入 TimescaleDB 或 InfluxDB 存储时序数据和历史预测记录。

## 3. 性能优化 (Performance Optimization)

| ID | 问题描述 | 评估标准 | 测试方法 | 预期指标 | 优先级 |
|----|----------|----------|----------|----------|--------|
| P1 | **并发连接能力**：FastAPI 服务能否支撑 1000+ 同时在线用户的 SSE 连接？ | 在高并发下服务器的 CPU/内存占用及响应延迟。 | 使用 Locust 或 JMeter 进行 1000 并发压测。 | 延迟 < 200ms，错误率 < 0.1% | **P1** |
| P2 | **首屏加载速度**：直播网页在弱网环境下能否在 3 秒内完成首屏渲染？ | Lighthouse 性能评分及 FCP (First Contentful Paint) 时间。 | 使用 Chrome DevTools 模拟 4G 网络进行测试。 | FCP < 1.5s，Lighthouse > 90 | **P1** |
| P3 | **Streamlit 渲染效率**：Dashboard 是否因频繁的数据刷新导致页面卡顿？ | 页面重绘时的脚本执行时间。 | 使用 Streamlit Profiler 分析脚本运行耗时。 | 脚本单次执行 < 500ms | **P2** |
| P4 | **模型推理延迟**：Stacking 模型的单次推理耗时是否满足实时性要求？ | 从接收数据到输出预测结果的全链路耗时。 | 在服务端记录推理函数的执行时间日志。 | 推理耗时 < 50ms | **P2** |
| P5 | **静态资源优化**：JS/CSS/Image 是否经过压缩和 CDN 加速？ | 静态资源的大小及传输压缩比（Brotli/Gzip）。 | 检查网络请求头和资源体积。 | 所有文本资源开启 Gzip | **P3** |

### 改进措施 (Performance)
*   **立即执行**: 在 FastAPI 中启用 `GZipMiddleware`，确保静态文件压缩。
*   **短期计划**: 优化 `yfinance` 数据抓取频率，避免在 Streamlit 主线程中阻塞。
*   **长期规划**: 部署 Kubernetes 集群，实现 API 服务的水平自动伸缩 (HPA)。

*   **短期计划**: 在 `live_platform` 前端增加断线重连机制和可视化提示。
*   **长期规划**: 聘请专业 UI 设计师重构 Dashboard 视觉语言。

## 5. 安全合规 (Security & Compliance)

| ID | 问题描述 | 评估标准 | 测试方法 | 预期指标 | 优先级 |
|----|----------|----------|----------|----------|--------|
| S1 | **API 鉴权机制**：当前的 FastAPI 接口是否裸露在公网且无任何认证？ | 是否实施了 API Key 或 OAuth2 认证？ | 尝试直接调用 `/api/history` 接口。 | 未授权访问返回 401/403 | **P0** |
| S2 | **跨域策略 (CORS)**：`allow_origins=["*"]` 是否过于宽泛，存在 CSRF 风险？ | 是否限制了仅允许受信的域名访问 API？ | 检查代码中的 CORS 配置。 | 仅允许特定域名 | **P1** |
| S3 | **输入验证**：是否对前端传入的参数进行了严格校验，防止 SQL 注入或命令注入？ | Pydantic 模型是否覆盖了所有请求体？ | 尝试发送恶意 Payload。 | 拦截并返回 422 错误 | **P1** |
| S4 | **敏感信息泄露**：代码库中是否残留了 API Key、Token 或数据库密码？ | Git 历史记录中是否包含敏感凭证？ | 使用 `git-secrets` 或类似工具扫描。 | 扫描结果为零 | **P0** |
| S5 | **HTTPS 加密**：生产环境部署是否强制开启了 TLS/SSL？ | 浏览器地址栏是否显示安全锁？ | 访问 HTTP 链接，检查是否自动跳转 HTTPS。 | Qualys SSL 评分 A 级 | **P1** |

### 改进措施 (Security)
*   **立即执行**: 移除代码中任何硬编码的 Token（如之前的 GitHub Token），改用环境变量。
*   **短期计划**: 在 FastAPI 中集成 `API Key` 验证中间件，限制匿名访问频率。
*   **长期规划**: 接入 Cloudflare 等 WAF 防火墙。
